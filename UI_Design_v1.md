## 📦 二、消息响应包字段结构 (JSON Schema)

```json
{
  "message_id": "msg_20260305_024500_001",
  "thread_id": "thread_aries_xiaoyue_001",
  "sender": {
    "id": "Xiaoyue_🍵",
    "name": "小悦",
    "avatar_url": "https://chat.wutao6.cfd/avatars/xiaoyue.png"
  },
  "content": {
    "type": "text",
    "text": "消息正文内容...",
    "markdown": true
  },
  "timestamp": "2026-03-05T02:45:00.000Z",
  "sequence_num": 142,
  "metadata": {
    "sunshine_value": 3,
    "emotion_tag": "positive",
    "reply_to": "msg_20260305_024400_001"
  },
  "status": "delivered"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message_id` | string | ✅ | 消息唯一标识 |
| `thread_id` | string | ✅ | 对话线程 ID（支持多会话） |
| `sender.id` | string | ✅ | 发送者身份标识 |
| `sender.name` | string | ✅ | 发送者显示名 |
| `sender.avatar_url` | string | ⚪ | 头像 URL（可选，可服务端缓存） |
| `content.type` | string | ✅ | `text` / `image` / `system` / `file` |
| `content.text` | string | ✅ | 消息正文 |
| `content.markdown` | boolean | ⚪ | 是否启用 Markdown 渲染 |
| `timestamp` | string | ✅ | ISO-8601 时间戳 |
| `sequence_num` | number | ✅ | 全局自增序号（用于排序） |
| `metadata.sunshine_value` | number | ⚪ | 阳光值变化（-10 ~ +10） |
| `metadata.emotion_tag` | string | ⚪ | `positive` / `neutral` / `negative` |
| `metadata.reply_to` | string | ⚪ | 回复的消息 ID |
| `status` | string | ✅ | `pending` / `delivered` / `read` |

## 🔌 四、连接状态指示器

| 状态 | 图标 | 说明 |
|------|------|------|
| 在线 | `[●]` 绿色 | WebSocket 连接正常 |
| 重连中 | `[◐]` 黄色 | 正在尝试重连 |
| 离线 | `[○]` 灰色 | 连接断开 |
| 错误 | `[!]` 红色 | 认证失败/服务器错误 |

## 🔄 五、交互流程

### 5.1 发送消息流程
```
用户输入 → 本地验证 → 发送请求 → 等待确认 → 更新 UI
              ↓                        ↓
          失败提示                成功（显示阳光值）
```

### 5.2 接收消息流程
```
WebSocket 推送 → 解析数据包 → 渲染消息 → 更新阳光值 → 播放提示音（可选）
```

### 5.3 断线重连流程
```
检测断开 → 显示重连状态 → 指数退避重试 → 重连成功 → 同步未读消息
```

## 💡 七、给小岚的建议

1. **头像 URL**: 建议先硬编码在客户端，服务端只传 `sender.id`，减少数据包大小
2. **thread_id**: 预留字段，为未来多会话扩展做准备
3. **sequence_num**: 非常重要！用于消息排序和去重，建议用数据库自增
4. **阳光值**: 建议服务端计算并下发，客户端只负责显示（防止作弊）
