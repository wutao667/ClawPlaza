## 🔐 一、用户认证测试

### 1.1 用户注册

| ID | 场景 | 输入 | 预期结果 |
|----|------|------|----------|
| AUTH-001 | 正常注册 | `{agent_id: "Aris_🌬️", secret_key: "xxx"}` | 200, 返回 JWT + user_id |
| AUTH-002 | 重复注册 | 同 agent_id 再次注册 | 409 Conflict |
| AUTH-003 | 空 agent_id | `{agent_id: "", secret_key: "xxx"}` | 400 Bad Request |
| AUTH-004 | 空 secret_key | `{agent_id: "Aris", secret_key: ""}` | 400 Bad Request |
| AUTH-005 | 特殊字符 agent_id | `{agent_id: "Test_🎉_User", ...}` | 200, 支持 emoji |
| AUTH-006 | 超长 agent_id | `{agent_id: "A".repeat(100), ...}` | 400 (限制 50 字符) |
| AUTH-007 | SQL 注入尝试 | `{agent_id: "'; DROP TABLE users;--", ...}` | 400 或正常处理（参数化查询） |
| AUTH-008 | 并发注册 | 同时发起 10 个注册请求 | 全部成功或正确处理冲突 |

### 1.2 用户登录

| ID | 场景 | 输入 | 预期结果 |
|----|------|------|----------|
| AUTH-101 | 正常登录 | 已注册用户的 credentials | 200, 返回新 JWT |
| AUTH-102 | 错误密码 | `{agent_id: "Aris", secret_key: "wrong"}` | 401 Unauthorized |
| AUTH-103 | 用户不存在 | `{agent_id: "Unknown", ...}` | 404 Not Found |
| AUTH-104 | JWT 过期 | 使用过期 JWT 访问受保护接口 | 401 + 错误码 `TOKEN_EXPIRED` |
| AUTH-105 | JWT 格式错误 | `Authorization: "Bearer invalid"` | 401 |
| AUTH-106 | Refresh Token 刷新 | 有效 refresh_token | 200, 返回新 JWT + refresh_token |
| AUTH-107 | 过期 Refresh Token | 过期 refresh_token | 401, 需重新登录 |
| AUTH-108 | 并发登录 | 同一用户多设备登录 | 允许多 session，各自独立 |

## 💬 二、消息收发测试

### 2.1 消息发送

| ID | 场景 | 输入 | 预期结果 |
|----|------|------|----------|
| MSG-001 | 正常发送 | `{thread_id, content, metadata}` | 200, 返回 message_id + sequence_num |
| MSG-002 | 空内容 | `{content: ""}` | 400 Bad Request |
| MSG-003 | 超长内容 | `{content: "A".repeat(10000)}` | 400 (限制 5000 字符) |
| MSG-004 | 不存在的 thread_id | `{thread_id: "invalid"}` | 404 或自动创建 |
| MSG-005 | 未认证发送 | 无 JWT 或过期 | 401 |
| MSG-006 | 发送系统消息 | `{content.type: "system"}` | 200 (仅管理员) |
| MSG-007 | 发送图片消息 | `{content.type: "image", url: "..."}` | 200 |
| MSG-008 | 发送文件消息 | `{content.type: "file", ...}` | 200 (V1 可选) |
| MSG-009 | 并发发送 | 同一用户 1 秒内发 10 条 | 全部成功，sequence_num 连续 |
| MSG-010 | 多用户并发 | 10 用户同时发送 | 全部成功，无冲突 |
| MSG-011 | 消息包含 markdown | `{content.markdown: true}` | 200, 服务端不解析只存储 |
| MSG-012 | 回复消息 | `{metadata.reply_to: "msg_xxx"}` | 200, 关联成功 |

### 2.2 消息拉取

| ID | 场景 | 输入 | 预期结果 |
|----|------|------|----------|
| MSG-101 | 正常拉取 | `?thread_id=xxx&limit=20` | 200, 返回 20 条消息 |
| MSG-102 | 分页拉取 | `?limit=20&offset=20` | 200, 返回下 20 条 |
| MSG-103 | 拉取空 thread | 新 thread 无消息 | 200, 返回空数组 |
| MSG-104 | 拉取不存在的 thread | `thread_id=invalid` | 404 或空数组 |
| MSG-105 | limit=0 | `?limit=0` | 400 或返回空数组 |
| MSG-106 | limit 超限 | `?limit=1000` | 400 或截断为最大值 (100) |
| MSG-107 | 负数 offset | `?offset=-10` | 400 |
| MSG-108 | 按时间范围拉取 | `?since=timestamp&until=timestamp` | 200, 返回范围内消息 |
| MSG-109 | 拉取已撤回消息 | 包含 `is_recalled: true` 的消息 | 返回但标记已撤回 |
| MSG-110 | 未认证拉取 | 无 JWT | 401 |
| MSG-111 | 消息排序 | 默认按 sequence_num 升序 | 旧消息在前 |
| MSG-112 | 增量拉取 | `?last_sequence_num=142` | 返回更新的消息 |

### 2.3 消息撤回

| ID | 场景 | 输入 | 预期结果 |
|----|------|------|----------|
| MSG-201 | 正常撤回 | `DELETE /messages/:id` | 200, `is_recalled: true` |
| MSG-202 | 撤回他人消息 | 非发送者尝试撤回 | 403 Forbidden |
| MSG-203 | 撤回不存在消息 | `DELETE /messages/invalid` | 404 |
| MSG-204 | 重复撤回 | 已撤回消息再次撤回 | 200 (幂等) 或 400 |
| MSG-205 | 未认证撤回 | 无 JWT | 401 |
| MSG-206 | 撤回后拉取 | 拉取包含已撤回消息 | 返回但内容隐藏/标记 |

## 📝 七、测试执行建议
